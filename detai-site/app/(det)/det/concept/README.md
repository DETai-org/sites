# Концепция DET (/det/concept)

Раздел `/det/concept` — это введение и ядро концепции DET. Он объясняет основу подхода делектико-экзестенциальной терапии и подводит к постулатам, которые раскрывают ключевые идеи.

## Роль постулатов

Постулаты — это **pillar pages**: они читаются как самостоятельные страницы и вместе образуют каркас смыслов.  
**Pillar page = глава книги в сжатом виде.**

Важно: постулат — это не “рубрика блога”, а **каноническая концептуальная ось** темы.

## Связка “постулат ↔ рубрика ↔ посты”

- **Рубрика блога** = вход в тему: поток, список постов, навигация по материалам.
- **Постулат** = каноническая страница: собирает и структурирует смысл темы.
- Связь двунаправленная:
  - рубрика **ссылается на постулат** (как на “концептуальное изложение”);
  - постулат **ссылается на рубрику** (как на “прикладной слой / каталог постов”).
- Внутри постулата текст по мере необходимости **ссылается на конкретные посты** из связанной рубрики (как на раскрытия, примеры и углубления).

Таким образом получается замкнутый контур навигации:
**пост → рубрика → постулат → (обратно к) постам**.

## Рубрики, не связанные с постулатами

Не все рубрики блога соответствуют постулатам концепции.

В частности, рубрика **«Скилл самопиара»** (routeSlugs":"ru": "skill-samopiara") не относится к концептуальному слою DET и не имеет связанного постулата. 
Она используется для мета-контента: описаний процессов, разборов того, как создавались проекты, тексты, идеи или продукты внутри экосистемы.

Такие посты фиксируют **путь, опыт и способ работы**, а не элементы концепции.  
Поэтому:
- у этой рубрики нет страницы постулата в `/det/concept`;


## Принцип “текст виден в репозитории”

Текст постулатов хранится прямо в файлах страниц `page.tsx` соответствующих постулатов (а не в динамических `[slug]` и не в отдельном data-layer). Это делает смысловые изменения прозрачными в Git и удобными для редактирования.

## Глоссарий (локальный)

- **Concept** — раздел концепции DET (`/det/concept`): “введение/ядро” + постулаты.
- **Постулат** — страница внутри `concept`, которая является pillar page и концептуальной осью темы.
- **Рубрика** — блоговый вход в тему (страница-агрегатор постов), связанная с постулатом.
- **Посты** — отдельные тексты, которые раскрывают элементы постулата и формируют “прикладной слой” темы.
