# Политика хранения обложек постов

## Цель

Эта политика описывает, где хранить обложки постов, если материалы публикуются одновременно на `personal-site` и `detai-site`.

## Основной принцип

- Если пост выходит **на обоих сайтах**, допустимо хранить изображение **в одном месте** — в `personal-site/public/images/posts/`.
- При этом в данных поста можно использовать:
  - локальный путь `src` для конкретного сайта (если изображение продублировано),
  - **или** прямую ссылку на ресурс персонального сайта, если используется единый источник.

## Рекомендации

1. **Источник истины** для общих изображений — `personal-site/public/images/posts/`.
2. В `detai-site/public/images/posts/` храните только те обложки, которые:
   - уникальны для DETai,
   - или временно продублированы для автономной сборки.
3. Если изображение используется на обоих сайтах и важно избежать дублирования, допустимо:
   - сослаться на личный сайт через абсолютный URL,
   - или поддерживать синхронизированную копию (например, вручную или через скрипт).

## Пример

Если обложка используется на обоих сайтах:

- **Источник**: `personal-site/public/images/posts/maxresdefault.jpg`
- **Данные поста**: `coverImage.src` может ссылаться на абсолютный URL личного сайта,
  либо на локальный путь, если файл продублирован в `detai-site/public/images/posts/`.
