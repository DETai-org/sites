## Кнопки Button.tsx

### **Функции кнопок (3 состояния)**

Каждая кнопка имеет три смысловые функции поведения:

* **Hover** — реакция при наведении.
* **Active** — реакция при нажатии.
* **Focus** — состояние при выделении клавиатурой.

### **Типы кнопок (5 уровней действий)**

| Typography role | Тип кнопки      | Стили                 | Имеет shimmer?            | Шрифты    | Существует |
| --------------- | --------------- | --------------------- | ------------------------- | --------- | ---------- |
| `font-sans`     | **Primary**     | самая важная, главная | ✅ Да                      | Open Sans | ✅          |
| `font-sans`     | **Secondary**   | вспомогательная       | ❌ Нет                     | Open Sans | ✅          |
| `font-sans`     | Tertiary        | текстовая             | ❌ Нет                     | —         | ❌          |
| `font-serif` или font-accent    | Ghost / Premium | (в будущем)           | может иметь другой эффект | —         | ❌          |
| `font-sans`     | Destructive     | красная / опасная     | ❌ точно нет               | —         | ❌          |

Примечание:
1.
Таблица отражает текущее состояние реализации кнопок.
Связь между Typography role и типом кнопки не является жёсткой и может быть пересмотрена.
Все кнопки являются UI-компонентами и по умолчанию относятся к интерфейсной типографике (font-sans) а у нас она =  Шрифту Open Sans
Использование font-serif в кнопках возможно только как осознанное исключение для специальных входных или ритуальных действий.

Связка с Tailwind (чтобы всё сошлось в голове)

font-sans → интерфейс / действие

font-serif → смысл / манифест

font-accent → декоративный акцент (и тогда м.б. font-accent → Great Vibes?)

2.
В системе предусмотрены три типографические роли (шрифта Lora, Open Sans, Great Vibes), из которых в кнопках сейчас используется только одна.
Также существует три ещё не реализованных типа кнопок — для них типографическая роль, визуальный стиль и анимационное поведение (например, shimmer) не зафиксированы и могут быть выбраны свободно: как уже используемые, так и любые другие.
3.
shimmer — это фирменная фича primary.

## Heading.tsx

* Единственный компонент заголовков для всего проекта.
* Поддерживает уровни `1 | 2 | 3` через проп `level`.
* Используй импорт `"@/components/ui/Heading"` или относительный путь из секций, чтобы исключить дублирование компонентов и расхождения в стилях.

Смотри, где используются эти элементы, в таблице выше в [components/README.md](../README.md).

## BodyText.tsx

* Базовый абзацный текстовый компонент с шрифтом Open Sans.
* Варианты:
  * `sectionDefaultDark` — базовый текст для секций на тёмном фоне (по умолчанию).
  * `sectionDefaultOnLight` — базовый текст для секций на светлом фоне.
  * `sectionBrand` — акцентный текст в секциях.
  * `projectCard` — компактный текст для карточек проектов (витрина): `text-mobile-base` + `leading-mobile-tight`, на десктопе `md:text-base`.
  * `infoCard` — текст для описательных карточек / блоков "для кого": `text-mobile-base` + `leading-mobile-normal`, на десктопе `md:text-base md:leading-relaxed`.
* Используй в секциях вместо локальных `<p>`, а в карточках — нужный карточный вариант, чтобы сохранять единый стиль body-текста.
